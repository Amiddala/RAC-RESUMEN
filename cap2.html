Switching — Cisco

Análisis técnico completo de switching en capa 2, VLANs, protocolo Spanning Tree y prevención de bucles en redes LAN.

📋 Capa OSI: Capa 2 (Enlace de Datos) 🔧 Tecnología: Ethernet / Cisco IOS 📄 Diapositivas: 14 slides procesadas

🔀Introducción al Switching

El switching es el proceso fundamental de mover tramas de datos entre hosts dentro de una red local. Opera en la Capa 2 (Enlace de Datos) del modelo OSI, utilizando direcciones MAC para tomar decisiones de reenvío.

🖥️
Definición
Mover tramas de un host a otro host, dentro de una red local (LAN). Opera exclusivamente mediante direcciones MAC.
🏢
Caso de Uso
Conectar múltiples dispositivos dentro de la misma red en una oficina o edificio, transportando datos con mínimas colisiones.
⚡
Ventaja Principal
Separa los dominios de colisión. Cada puerto del switch es un dominio de colisión independiente.
🤖
Operación Automática
El switch gestiona automáticamente el envío y recepción de tramas sin intervención del administrador para el tráfico normal.
Nota técnica OSI:

El switching opera en Capa 2. A diferencia del routing (Capa 3 / IP), el switch no analiza paquetes IP — solo examina las direcciones MAC de origen y destino de las tramas Ethernet.

📚Conceptos Clave

¿Por qué se usan los Switches?

  • 1 Segmentación de colisiones: Cada puerto del switch constituye su propio dominio de colisión, eliminando las colisiones presentes en las topologías de hub.
  • 2 Forwarding inteligente: El switch aprende las direcciones MAC de cada dispositivo conectado y construye una tabla CAM (Content Addressable Memory). Solo envía la trama al puerto de destino correcto.
  • 3 Transparencia: Los hosts no necesitan configuración especial. El switch opera de forma transparente entre ellos.
  • 4 Comunicación interna LAN: Ethernet (Capa 2) es el protocolo por excelencia para comunicar hosts LAN desde el router hacia adentro. Es el medio de transporte que los switches utilizan para comunicarse entre sí.

Dominio de Colisión vs Dominio de Broadcast

Concepto Dispositivo que delimita Descripción
Dominio de Colisión Switch (por puerto) Segmento donde dos transmisiones simultáneas causan colisión. El switch aísla cada puerto.
Dominio de Broadcast Router Área donde un broadcast se propaga a todos los dispositivos. El switch NO lo limita (sin VLANs).

📦Trama Ethernet

Ethernet es el protocolo de Capa 2 utilizado para la comunicación entre hosts LAN. El switch procesa y reenvía estas tramas basándose en las direcciones MAC.

Estructura de la Trama Ethernet II

# Campo Tamaño Descripción
1 Preamble 7 bytes Sincronización. Patrón alternado de 1s y 0s para sincronizar el receptor.
2 SFD (Start Frame Delimiter) 1 byte Indica el inicio real de la trama (valor 0xAB).
3 Destination (MAC Destino) 6 bytes Dirección MAC del host o dispositivo de destino. El switch usa este campo para el reenvío.
4 Source (MAC Origen) 6 bytes Dirección MAC del emisor. El switch aprende la tabla CAM de este campo.
5 Type / EtherType 2 bytes Protocolo de Capa 3 encapsulado (0x0800 = IPv4, 0x0806 = ARP, 0x86DD = IPv6).
6 Data and Pad 46–1500 bytes Payload (paquete IP encapsulado). Mínimo 46 bytes; si el dato es menor, se rellena con padding.
7 FCS (Frame Check Sequence) 4 bytes CRC-32 para detección de errores de transmisión.
⚠️ Diferencia crítica con IP:

Las tramas Ethernet NO poseen un campo TTL (Time to Live) como los paquetes IP. Esto significa que una trama que entra en un bucle de Capa 2 puede circular indefinidamente, a diferencia de un paquete IP que será descartado al llegar a TTL=0.

Tamaño total de la trama

Tamaño Mínimo
64 bytes (sin preámbulo/SFD). Garantiza detección de colisiones en CSMA/CD.
Tamaño Máximo (MTU)
1518 bytes (sin preámbulo/SFD). El campo de datos tiene un máximo de 1500 bytes.

🗂️VLANs — Virtual LANs

Las VLANs permiten la segmentación lógica de la red sobre una infraestructura física compartida. Solucionan el problema de tener la misma red física en cada edificio o piso.

Definición técnica

Concepto:

Las VLANs permiten separar en dominios lógicos los enlaces. Es decir, permiten hacer enlaces lógicos de un enlace físico. Un switch con VLANs puede pertenecer a múltiples dominios de broadcast independientes.

Problema que resuelven (Topología LAN clásica)

En la topología LAN anterior, cada edificio tenía la misma red física replicada en cada piso:

🏠
LAN 1 (5° piso)
Dormitorios de estudiantes. Red independiente por piso en cada edificio.
🏫
LAN 2 (3° piso)
Oficinas del cuerpo docente. Red física separada, sin comunicación lógica entre edificios.

Beneficios de las VLANs

Beneficio Descripción Técnica
SeguridadAísla el tráfico entre grupos (docentes, estudiantes, invitados).
Reducción de costosUna sola infraestructura física soporta múltiples redes lógicas.
Superior rendimientoReduce el tamaño de los dominios de broadcast, disminuyendo tráfico innecesario.
Mitigación de tormenta de broadcastLos broadcasts no se propagan entre VLANs.
Mayor eficiencia del personal TIAdministración centralizada y agrupación lógica por función.
Administración de aplicaciones simplificadaPermite agrupar usuarios por proyecto o aplicación independientemente de la ubicación física.

Ejemplo de asignación VLAN

Configuración VLAN típicaCISCO IOS
! Crear VLANs en el switch
Switch(config)# vlan 10
Switch(config-vlan)# name CUERPO_DOCENTE
Switch(config)# vlan 20
Switch(config-vlan)# name ESTUDIANTE
Switch(config)# vlan 30
Switch(config-vlan)# name INVITADO

! Asignar puerto a VLAN (Access mode)
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

! Configurar puerto Trunk (transporta múltiples VLANs)
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30

🌲Protocolo Spanning Tree (STP)

Definición oficial:

El protocolo Spanning Tree tiene como función gestionar la presencia de bucles en topologías de red debido a la existencia de enlaces redundantes (necesarios en muchos casos para garantizar la disponibilidad de las conexiones). El protocolo permite a los dispositivos de interconexión activar o desactivar automáticamente los enlaces de conexión, de forma que se garantice la eliminación de bucles.

Objetivo dual del STP

✅
Permite redundancia
Permite tener múltiples rutas físicas entre dispositivos. Si una ruta falla, el enlace bloqueado puede activarse automáticamente.
🚫
Previene bucles
Bloquea los puertos redundantes para que lógicamente solo exista una ruta activa entre dos dispositivos en la topología.

Redundancia de Capa 2

La redundancia de Capa 2 mejora la disponibilidad de la red con la implementación de rutas de red alternas mediante el agregado de equipos y cables. La interrupción de una sola ruta no genera impacto en la conectividad de los dispositivos en la red.

🔁Bucles de Capa 2

Condición de bucle:

Cuando existen varias rutas entre dos dispositivos en la red y STP se ha deshabilitado en los switches, puede generarse un bucle de Capa 2.

Por qué los bucles son peligrosos

  • 1 Sin TTL en Ethernet: Las tramas Ethernet no poseen un campo TTL (Time to Live) como los paquetes IP. Una trama atrapada en un bucle circulará indefinidamente sin ser descartada automáticamente.
  • 2 Broadcast flooding: Las tramas de broadcast se envían a todos los puertos del switch, excepto el puerto de origen. Esto asegura que todos los dispositivos del dominio de broadcast reciban la trama. Con múltiples rutas, se genera un bucle sin fin.
  • 3 Inestabilidad de tabla MAC: El switch recibe la misma trama por múltiples puertos y actualiza constantemente la dirección MAC en su tabla, causando inestabilidad (MAC flapping).

⛈️Tormenta de Broadcast

Definición:

Una tormenta de broadcast se produce cuando existen tantas tramas de broadcast atrapadas en un bucle de Capa 2 que el ancho de banda disponible se agota completamente.

📡
Causa
Bucles de Capa 2 sin STP activo. Las tramas broadcast se replican continuamente en cada iteración del bucle.
💀
Consecuencia
No existe ancho de banda disponible para el tráfico legítimo. La red queda completamente no disponible para la comunicación de datos.
🔮
Inevitabilidad
La tormenta de broadcast es inevitable en una red con bucles de Capa 2 sin mecanismo de prevención.

Flujo de una tormenta de broadcast

  • 1 Un host envía un frame broadcast (ej: ARP Request). El switch lo reenvía por todos sus puertos (excepto origen).
  • 2 El frame llega a otro switch que también lo reenvía por todos sus puertos, incluyendo el camino de regreso al primer switch.
  • 3 Sin TTL y sin STP, el frame sigue circulando, duplicándose con cada iteración hasta saturar completamente el ancho de banda de todos los enlaces.
  • 4 La CPU de los switches se satura procesando los frames. La red colapsa totalmente (denial of service a nivel L2).

🧮Algoritmo STA (Spanning Tree Algorithm)

STA es el algoritmo que STP utiliza para determinar la topología libre de bucles. Selecciona un Root Bridge y calcula los caminos más eficientes hacia él.

Campos del BID (Bridge ID)

Cada switch STP tiene un Bridge ID (BID) compuesto por:

Campo Tamaño Descripción
Priority 2 bytes Valor de prioridad del bridge. Rango: 0–65535. Valor por defecto: 32768. El switch con menor prioridad gana la elección de Root Bridge.
MAC Address 6 bytes Dirección MAC del switch. Se usa como desempate cuando dos switches tienen la misma prioridad. La MAC más baja gana.

Ejemplo del diagrama (valores del material)

Switch Priority MAC Address Rol STP
S1 24577 000A00333333 Root Bridge
S2 32769 000A00111111 Non-Root
S3 32769 000A00222222 Non-Root
¿Por qué S1 es el Root Bridge?

S1 tiene la prioridad más baja (24577) vs S2 y S3 (ambos con 32769). En STP, menor prioridad = mayor preferencia. El Root Bridge es el switch con el BID más bajo.

Tipos de puertos STP

🟢
Root Port
Puerto con el camino de menor costo hacia el Root Bridge. Uno por switch no-root. Estado: Forwarding.
🔵
Designated Port
Puerto con el mejor camino hacia el Root Bridge para un segmento. Uno por segmento. Estado: Forwarding.
🔴
Non-Designated Port
Puerto bloqueado para prevenir bucles. No reenvía tramas de usuario. Estado: Blocking. Solo recibe BPDUs.

Proceso de STA en 4 pasos

  • 1 Elegir Root Bridge: Todos los switches intercambian BPDUs. El switch con menor BID se convierte en Root Bridge. Todos sus puertos quedan en estado Forwarding (Designated Ports).
  • 2 Elegir Root Ports: Cada switch no-root selecciona el puerto con menor costo de ruta acumulado hacia el Root Bridge como su Root Port.
  • 3 Elegir Designated Ports: En cada segmento de red, el switch con el menor costo de ruta hacia el Root Bridge tiene su puerto como Designated Port (Forwarding).
  • 4 Bloquear Non-Designated Ports: Todos los puertos que no son Root ni Designated quedan en estado Blocking, eliminando los bucles lógicamente.

Q1 ¿En qué capa del modelo OSI opera el switching y qué tipo de dirección utiliza para tomar decisiones de reenvío?
  • A Capa 3, utilizando direcciones IP
  • B Capa 2, utilizando direcciones MAC
  • C Capa 1, utilizando señales eléctricas
  • D Capa 4, utilizando puertos TCP/UDP
✅ Correcto. El switching opera en la Capa 2 (Enlace de Datos) del modelo OSI. Utiliza direcciones MAC (48-bit) para construir la tabla CAM y reenviar tramas al puerto correcto.
Q2 ¿Cuál es el tamaño exacto del campo "Data and Pad" en una trama Ethernet II y por qué existe un mínimo?
  • A 0 – 1500 bytes; el mínimo no existe
  • B 64 – 1500 bytes; para compatibilidad con IPv6
  • C 46 – 1500 bytes; el mínimo de 46 bytes garantiza la detección de colisiones en CSMA/CD
  • D 100 – 1500 bytes; por requisitos del protocolo ARP
✅ Correcto. El campo Data and Pad tiene entre 46 y 1500 bytes. El mínimo de 46 bytes (junto con los headers, hace una trama mínima de 64 bytes) es necesario para que la colisión sea detectable en CSMA/CD antes de que el primer bit llegue al extremo más lejano del segmento.
Q3 En el escenario del material, hay tres switches: S1 (Priority=24577, MAC=000A00333333), S2 (Priority=32769, MAC=000A00111111) y S3 (Priority=32769, MAC=000A00222222). ¿Cuál es el Root Bridge y por qué?
  • A S1, porque tiene la prioridad más baja (24577)
  • B S2, porque tiene la MAC más baja (000A00111111)
  • C S3, porque tiene la prioridad más alta (32769)
  • D Todos son Root Bridge simultáneamente
✅ Correcto. S1 es el Root Bridge porque tiene la prioridad BID más baja (24577 < 32769). En STP, el switch con el BID numéricamente más bajo gana la elección. La MAC solo se usa como desempate cuando la prioridad es igual.
Q4 ¿Por qué las tramas Ethernet en un bucle de Capa 2 son más peligrosas que los paquetes IP en un bucle de Capa 3?
  • A Porque Ethernet usa más ancho de banda que IP
  • B Porque las tramas Ethernet no tienen campo TTL, por lo que circularán indefinidamente sin ser descartadas
  • C Porque los switches son más lentos que los routers para procesar paquetes
  • D Porque Ethernet no soporta broadcast
✅ Correcto. Los paquetes IP tienen un campo TTL que se decrementa en cada salto y son descartados cuando TTL=0. Las tramas Ethernet no tienen TTL, por lo que una trama en un bucle L2 circulará para siempre, causando una tormenta de broadcast que satura el ancho de banda.
Q5 ¿Cuál es la función técnica de las VLANs en relación con los dominios de broadcast?
  • A Las VLANs aumentan el tamaño del dominio de broadcast para mayor alcance
  • B Las VLANs eliminan completamente los dominios de broadcast de la red
  • C Las VLANs crean dominios de broadcast lógicamente separados sobre una misma infraestructura física, segmentando el tráfico entre grupos
  • D Las VLANs solo funcionan a nivel de Capa 3 con enrutamiento
✅ Correcto. Las VLANs separan en dominios lógicos los enlaces físicos. Permiten hacer enlaces lógicos de un enlace físico. Un broadcast en VLAN 10 no se propaga a VLAN 20 o VLAN 30. Esto reduce el tráfico innecesario y mejora la seguridad, aunque sigue requiriendo un router o switch L3 para comunicación inter-VLAN.
